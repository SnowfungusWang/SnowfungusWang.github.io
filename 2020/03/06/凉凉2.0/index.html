4. 错题本

  1. 等等他问的是哪个数据库连接？？？？

  2. 表连接的方式(JOIN)：就是在笛卡尔积里筛

    • 内连接：
      • 两个表的属性会直接拼接起来。
      • 没有条件就是笛卡尔积
      • 包括：
        • 等值连接：连接条件是等号，找条件的交集。
          • 包括重复列
          • SELECT * from Table_A A JOIN Table_B B ON A.id = B.id
        • 不等连接：一个个做比较，所有满足条件的全部返回。
        • 自然连接：删除连接表中的复列。
    • 外连接：能匹配就匹配，不能匹配就取空
      • 分左右
      • 匹配条件是匹配的和全部的左/右
      • 另一个里没有的用空值填充
    • 交叉连接：不带where，返回笛卡尔积。（就是内连接不带条件）
    • 在连接查询中，通常不用 WHERE 而是用 ON，因为 WHERE 没有 ON 效率高。

  3. 数据库连接（好多，当听一乐了）：

    • ODBC即开放式数据库互连
      • 本质上是一组数据库访问API（应用程序编程接口），由一组函数调用组成，核心是SQL语句
    • OLE DB即数据库链接和嵌入对象（Object Linking and Embedding DataBase）。
      • 提供一种相同的访问接口，使得数据的使用者(应用程序)可以使用同样的方法访问各种数据，而不用考虑数据的具体存储地点、格式或类型
    • JDBC（Java Data Base Connectivity）一组Java语言编写的用于数据库连接和操作的类和接口，可为多种关系数据库提供统一的访问方式。通过JDBC完成对数据库的访问包括四个主要组件：Java应用程序、JDBC驱动器管理器、驱动器和数据源。

  4. 散列表（Hash table，也叫哈希表），是根据键（Key）而直接访问在内存储存位置的数据结构。它通过计算一个关于键值的函数，将所需查询的数据映射到表中一个位置来访问记录，这加快了查找速度。

    • 散列函数：直接定址、数字分析、平方取中、折叠、随机数、取模
    • 处理冲突：开放定址（探测）、单独链表、双散列、再散列

  5. 翻转链表：

    • 递归：遍历至结尾，再往前
    • 非递归：多加一个存储

  6. 多态的实现：

    • Java：方法表，指向具体的方向地址。
      • 存在优化
    • C++：虚函数表
