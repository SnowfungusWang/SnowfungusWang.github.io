【F-知识】http缓存

缓存规则

缓存规则信息包含在响应header里。

根据是否需要重新向服务器发起请求来分类，有强制缓存和对比缓存。客户端都是先向浏览器的缓存数据库请求。

  • 强制缓存：向缓存数据库请求到了就不再向服务器发请求。
  • 对比缓存：向缓存数据库请求标识，再向服务器请求验证数据是否失效。

两类规则可以同时生效，强制缓存优先级高。

强制缓存

响应header中会有两个字段来标明失效规则：Expires/Cache-Control。

  1. Expires：值是服务端返回的到期时间，相当于保质期。
    • http1.0的东西了，现在1.1基本不用了。
    • 到期时间是服务端生成的，可能和客户端有时间差
  2. Cache-Control：常见的取值有private、public、no-cache、max-age，no-store，默认为private。
    • private：客户端可以缓存。
    • public：客户端和代理服务器都可以缓存。（前端透明）
    • max-age：失效时间（单位是秒）。
    • no-cache：需要使用对比缓存来验证缓存数据。
    • no-store：所有内容都不会缓存，强制缓存，对比缓存都不会触发。

对比缓存

浏览器第一次请求数据时，服务器会将缓存标识与数据一起返回给客户端，客户端将二者备份至缓存数据库中。
再次请求数据时，客户端将备份的缓存标识发送给服务器，服务器根据缓存标识进行判断，判断成功后，返回304状态码，通知客户端比较成功，可以使用缓存数据。

  • 报文大小和请求时间减少，因为只返回header。
  • 缓存标识的传递：Last-Modified / If-Modified-Since
    • Last-Modified：服务器在响应请求时，告诉浏览器资源的最后修改时间。
    • If-Modified-Since：再次请求服务器时，通过此字段通知服务器上次请求时，服务器返回的资源最后修改时间。服务器拿这个来看是不是过期了。

【LC】BFS、DFS

Posted on 2020-03-14

1. 思路总结

BFS：

  • 找个队列，队头出，队头的结果往队尾里push。其实就是拿这个队列记录你待往下搜的下一层，所以增长是指数级别的。
  • 不会爆栈
  • 用来找最短路径、最小啥啥啦
  • 注意和图的结合

DFS

  • 空间和深度成正比
  • 有爆栈风险
  • 不能搜最短、最小。一般是把整个图整了。

C++默认4m栈，害。大概10w层。

2. 题目

  • 111：找二叉树的最小深度。

    • DFS
    • 根的深度为左右子树中较小的+1，然后递归去找。
    • 递归结束：当前节点不存在，返回0
    • 特殊情况：当前节点的左节点或者右节点不存在，返回

  • 279：给n，用最少的平方数的和表示他。（13=4+9，返回2）

    • BFS，最少问题
    • 看成图，最开始是0，结束是n，靠路径转移，找最短路径。
    • 队列放当前到的点，没空或者没到n就一直往下走。数组记录顶点的值。

  • 733： Flood Fill，洪水灌溉。在网格图中搜出联通块。

    • DFS和BFS都行。

    • DFS好写。

    • DFS的上下左右先写出来（方向向量）

      int dx[4]={}; int dy[4]={}省的写4个if

    • 边界判断可以做在调用函数前或者进入逻辑之前。

    • 注意不要走回头路，否则会死循环。

  • 200：算孤岛。统计联通块数目。

    • 遍历整个数组，遇到一个1就把当前的联通块变成0。
    • DFS：void DFS(int[][] grid, intx, int y)

  • 130：围棋吃子，4个方向包围就行。

    • 先把活的子标出来，然后把死的吃掉，再把活的恢复。
    • 活子：和边界联通的，所以只要对四条边上的活子DFS就行。

Event Loop

环图思考法

核心任务一直转，js脚本和css渲染在两边，一边执行完，才能去另一边。

css中：RAF（请求动画帧）的处理是最先的，把动画任务打包在一起。在css绘制和渲染之前，嵌套的会在下一帧执行。（Edge和Safari是在后面）。

微任务：会发生在js栈执行空了的时候，一次全执行空了，执行权一旦在微任务栈里，就不会停了。Promise、异步、setTImeOut……属于微任务。

  • 任务栈：一次一个
  • RAF：执行完当前所有，嵌套的下一次
  • 微任务：一直搞。
